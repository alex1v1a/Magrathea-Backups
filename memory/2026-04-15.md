# Memory Log - 2026-04-15

## Home Assistant Fix (23:25 CDT)

### Problem
Home Assistant was DOWN due to WSL2 mirrored networking issue. Port 8123 was not accessible from Windows even though the container was running.

### Root Cause
WSL2 `mirrored` networking mode was not working properly - the eth0 interface was DOWN and wouldn't come up.

### Solution Applied
1. Changed WSL networking mode from `mirrored` to `NAT` in `~/.wslconfig`
2. Restarted WSL to apply the new networking configuration
3. Verified eth0 interface came up with IP 172.26.117.82
4. Configured Windows port forwarding: `netsh interface portproxy add v4tov4 listenport=8123 listenaddress=0.0.0.0 connectport=8123 connectaddress=172.26.117.82`
5. Verified Home Assistant is now accessible on http://localhost:8123

### Result
✅ Home Assistant is now UP and responding on port 8123

### Configuration Changes
- `~/.wslconfig`: Changed `networkingMode=mirrored` to `networkingMode=NAT`
- Windows port proxy: Forwarded port 8123 to WSL IP

### Verification
```
Test-NetConnection localhost:8123 = True
HTTP response: Home Assistant HTML page served correctly
```
