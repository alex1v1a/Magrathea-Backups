1. Disabled legacy tasks: `DiscordSpeechBot`, `TrillianWatchdog`
2. Migrated to WSL (Linux) runtime for voice bot:

3. Patched `discord-ext-voice-recv` router to ignore corrupted stream errors (preventing crashes)
4. Removed auto-join code from `bot.py` (manual `!join` only)
5. Created WSL runner script with auto-restart loop
6. Updated scheduled task `TrillianVoiceBotWSL` with unlimited execution time

**Current Status:** Bot running stable in WSL, connects to Discord successfully, responds to `!join` command. Logs show "Trillian Voice#9916 ready" without forced voice connections.
